# NotebookLM — FHIR en Google Cloud

> Paso obligatorio. Consolida el tema en un cuaderno de NotebookLM (audio, preguntas y mapa mental) antes de cerrarlo.

## Pasos

- [ ] Abrir [notebooklm.google.com](https://notebooklm.google.com) con tu cuenta Google.
- [ ] Crear el cuaderno **FHIR — FHIR en Google Cloud**.
- [ ] Subir la lectura del tema como fuente (botón "Exportar para NotebookLM").
- [ ] Añadir como fuentes: la página de Cloud Healthcare API, los how-tos de FHIR stores, la capa gratuita de GCP y Synthea.
- [ ] Generar el Audio Overview y escucharlo una vez completa.
- [ ] Responder un examen oral de 10 preguntas sin mirar el material.

## Prompts sugeridos

- "Hazme un examen de troubleshooting: para cada código (401, 403, 404, 412) dame un escenario en Cloud Healthcare API y pídeme diagnóstico y solución."
- "Redacta un runbook de sesión de laboratorio: montar store R4, importar Synthea desde GCS, exportar a BigQuery y limpieza total a $0."
- "Explica con casos nacionales cuándo activar enableUpdateCreate y cuándo desactivar la integridad referencial, y qué riesgos se aceptan."
